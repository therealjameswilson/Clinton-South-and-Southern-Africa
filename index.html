Onsite Research

Clinton Library Pull Strategy

The first onsite pass should convert broad Clinton Digital Library hits into document-level evidence. Pull South Africa and Angola policy files first, then Zimbabwe, regional security, and health-policy files, with speechwriting and public framing records used only to explain public diplomacy.

South Africa Transition Core

Queue Mandela, de Klerk, Mbeki, Holbrooke, Lake, Berger, and NSC Africa files around recognition, democratic transition, bilateral visits, and post-apartheid economic/security cooperation.

Angola Peace and Sanctions

Second Pull

Mozambique, Landmines, and Demining

Before Travel

Pre-Request and Log

Ask staff which OA/ID folders can be pre-pulled, whether item scans exist, and what citation, reproduction, and declassification limits apply.

Full pull queue
Photograph the trail first. Folder title, collection path, item page, withdrawal sheet, first page, and last page.
Promote only document-level items. Record actual date, participants, page span, markings, release status, and source-note path.
Keep the same-day decision log. Include candidate, context only, duplicate/source-control, boundary review, or no action.

FRUS Production Standard

Compiler Workbench

Document Selection Test

Select records that show major U.S. foreign policy decisions and significant diplomatic activity. Keep include, context, exclude, boundary-review, and pending decisions explicit.

Production reference

Chronological Placement

Sort memcons and telcons by conversation date and Washington time. Treat trip books, schedules, and diary entries as controls until the substantive record is located.

Fields: date, washingtonTime, sortDate, placementNote

Source Note Drafting

Draft the first note in FRUS order: repository trail, collection or series, exact file or item identifier, markings, then transmission, meeting, drafting, clearance, approval, or marginalia details.

Fields: sourceNote, source.path, documentMarkings, handlingMarkings, clearance

Declassification Accounting

Track full releases, excisions, documents withheld in full, pending review, and missing scans. Do not let a public event page substitute for a released policy document.

Fields: declassificationStatus, withheldMaterial
Selection Major decision, significant diplomatic activity, or supporting context.
Source Repository, collection, series, folder, item identifier, and markings.
Editorial Heading, date line, subject line, bracket/excision notes, and annotation.
Access Index terms, people, countries, related documents, and public statements.

Citation Standard

Make footnote 1 do the archival work.

Source notes should begin with Source:, move from repository to file path, then add classification and handling markings before editorial details such as channel, drafting, clearance, approval, attachments, handwritten notes, or omitted material.

  1. Repository and collection path
  2. File unit, folder, document ID, NAID, or case number
  3. Original classification, precedence, and handling markings
  4. Drafted by, cleared by, approved by, channel, or meeting location
